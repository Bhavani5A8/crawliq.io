SEO STANDARDS

What Good SEO Meta Tags Look Like

Title tags, meta descriptions, and H1s are the first signals Google reads. Getting them right is the foundation of any technical SEO evaluation.

T

Title Tag

50–60 characters. Place your primary keyword as close to the start as possible. End with the brand name separated by a dash or pipe. Every page must have a unique title — duplicate titles are one of the most common issues CrawlIQ surfaces during a full-site evaluation. Titles shorter than 30 characters leave impression potential on the table; over 60 characters get truncated in desktop SERPs.

Good Free AI SEO Audit Tool — CrawlIQ
Avoid Home | Welcome to Our Website
M

Meta Description

120–160 characters. Google doesn't use the meta description as a ranking signal directly, but it controls click-through rate. Include your primary keyword naturally (Google bolds matching terms), describe the page's value clearly, and end with a call-to-action. Descriptions below 70 characters are too thin; over 165 get rewritten by Google with arbitrary body copy — usually worse than what you wrote.

Good Free technical SEO evaluation reference. Assess 50+ on-page and technical signals per page. Results include structured remediation notes.
H1

H1 Heading

One per page. The H1 is the primary on-page relevance signal. It should contain your target keyword but not be identical to the title tag — search engines treat them as independent signals, so duplicate title/H1 is a missed opportunity. Keep H1 under 70 characters. Use H2–H3 tags to structure subtopics, not for visual styling. CrawlIQ flags pages with missing H1s, multiple H1s, and H1/title duplication in the crawler report.

🔗

Canonical Tag

Self-referencing on every indexable page. The canonical tag tells Google which URL is the authoritative version when duplicate or near-duplicate content exists across multiple URLs (e.g. ?utm_source= parameters, HTTP vs HTTPS, trailing slash variants). Missing canonicals on paginated content or filtered category pages is one of the most common technical debt items in e-commerce sites. CrawlIQ checks canonical presence, self-reference validity, and cross-domain canonical chains.

20-POINT CHECKLIST

SEO Assessment Checklist You Can Use Today

Work through each item as you assess your pages. Use CrawlIQ to automate the technical checks.

On-Page Basics

  1. Title tag is 50–60 characters. Not "Home" or "Welcome". Contains your primary keyword.
  2. Meta description is 120–160 characters. Has a clear call-to-action. Not duplicated from title.
  3. One H1 per page. Contains the primary keyword. Not the same as title tag.
  4. H2–H3 headings structure the content. Logical hierarchy, not used just for visual styling.
  5. Target keyword appears in the first 100 words. Natural usage — not keyword stuffed.

Technical SEO

  1. Canonical tag is correct. Points to the preferred URL. No self-referencing loops.
  2. Page returns HTTP 200. No accidental 404s or redirect chains to orphan URLs.
  3. HTTPS is enforced. HTTP version redirects to HTTPS. No mixed content.
  4. robots.txt permits evaluation access. No accidental Disallow: / on production.
  5. sitemap.xml is submitted to Google Search Console. Updated automatically. No 404 URLs in the sitemap.

Content Quality

  1. Page has at least 300 words of body content. Thin content ranks poorly. Expand or consolidate pages.
  2. Content answers the search intent. Informational? Transactional? Navigational? Match it.
  3. No duplicate content across pages. Verify with a site: search or CrawlIQ evaluation.
  4. Images have descriptive alt text. Not "image1.jpg" or empty strings. Keyword where relevant.
  5. Content is updated within the last 12 months. Stale content gets de-prioritised by Google.

Links & Authority

  1. Internal links use descriptive anchor text. Not "click here". Use the keyword or page topic.
  2. No broken internal links (4xx). Use a site evaluation to identify and address them.
  3. External links open in new tab and have rel="noopener". Security and UX best practice.
  4. At least 3 internal links point to this page. Orphan pages receive minimal search engine attention.
  5. Schema markup is present (Article, Product, FAQ, etc.). Eligible for rich results in Google SERPs.

Common questions about the CrawlIQ technical SEO evaluation reference

psychologyHow does CrawlIQ's AI SEO evaluation work?
CrawlIQ examines each page of a website using an async BFS evaluation engine and sends each page's signal data to an AI model — Groq (Llama 3), Gemini, OpenAI, or Claude (configurable). The AI evaluates content quality, keyword alignment, readability, and semantic relevance, then produces structured remediation recommendations ranked by impact. Output includes optimised title tags, meta descriptions, H1 rewrites, and content guidance.
bug_reportWhat SEO issues does CrawlIQ identify?
CrawlIQ identifies 15+ issue categories: missing or oversized title tags, missing meta descriptions, missing or multiple H1 tags, no H2 structure, canonical tag issues, duplicate meta descriptions across pages, pages returning 4xx/5xx errors, thin content (under 300 words), images missing alt text, HTTP instead of HTTPS, and URL structure issues like underscores or excessive depth.
downloadWhat data formats can I export?
CrawlIQ exports structured Excel (.xlsx) reference documents: a Full SEO Report with all evaluated pages, issues, and AI remediation guidance; a Per-Field Issues Report; a Remediation Table with optimised content; and a Technical SEO Assessment with per-page scores (0–100), letter grades (A–F), indexability status, and component-level breakdowns.

Creating a free account raises this to 200 pages per month with project saving and position tracking. Pro and Agency plans are available for larger sites. The evaluation engine uses breadth-first search from your root URL, follows internal links, and handles SSL failures with automatic HTTP fallback.
lockIs my website data secure and private?
Yes. All evaluation data is processed in isolated in-memory sessions and never stored permanently or shared. Data exists only for the duration of your session and is cleared when a new evaluation begins. CrawlIQ is fully open-source on GitHub and hosted on HuggingFace Spaces — you can review every line of code.
javascriptDoes CrawlIQ support JavaScript-rendered sites?
CrawlIQ uses an aiohttp-based async evaluation engine with BeautifulSoup for HTML parsing. It works best with server-side rendered, static, and WordPress sites, extracting titles, meta tags, headings, canonicals, Open Graph tags, body text, and image alt texts. For heavily client-side SPAs, enabling server-side rendering gives the best evaluation coverage.
