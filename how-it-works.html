How CrawlIQ Works

A step-by-step walkthrough of the 5-stage AI technical SEO audit process — from URL input to downloadable Excel report.

The 5-Step Audit Process

1

Enter Your Website URL

Navigate to the CrawlIQ App and paste any website URL into the audit input field. You can configure:

  • Page limit — 1 to 50 pages without login; up to 200 with a free account
  • Crawl depth — how many link levels deep to follow from the root URL
  • AI provider — Groq (default, fastest), Gemini, Claude, OpenAI, or rules-based fallback

No credit card or signup is required for the first 50 pages.

2

Async BFS Crawler Fetches Your Pages

CrawlIQ's backend crawler uses Python aiohttp to fetch all pages concurrently using breadth-first search (BFS) from your root URL. For each page it extracts:

  • Title tag, meta description, canonical URL
  • H1, H2, H3 heading structure and all heading text
  • Open Graph and Twitter Card meta tags
  • All internal links (to discover the next BFS layer)
  • Image alt attributes and image count
  • Body text word count and keyword frequency
  • HTTP status code, redirect chain if any, and final URL

SSL failures fall back to HTTP automatically so the crawl never stops on certificate errors. Real-time progress is streamed to your browser via Server-Sent Events (SSE).

3

Issue Detection and Page Scoring

Every crawled page is run through 50+ checks across 15 issue categories. Each page receives a score from 0–100 and an A–F grade. Issues checked include:

  • Title tag missing, too short (<30 chars), too long (>60 chars), or duplicated
  • Meta description missing, too short, too long, or duplicated across pages
  • H1 missing, multiple H1 tags, or H1 identical to title
  • Canonical absent, incorrect, or forming a redirect loop
  • Pages accidentally marked noindex or blocked by X-Robots-Tag
  • Broken links (4xx) and server errors (5xx) within internal links
  • Redirect chains consuming crawl budget
  • Images missing alt text
  • Thin content pages with under 300 words
  • Non-HTTPS pages or mixed content warnings
4

AI Analysis Generates Paste-Ready Fixes

After the crawl completes, select individual pages or run AI analysis on all results at once. CrawlIQ builds a structured prompt for each page containing its current SEO signals and detected issues, then sends it to your chosen AI model.

Each AI response includes:

  • Optimised title tag (within 50–60 character target)
  • Rewritten meta description (within 140–155 character target)
  • Improved H1 tag aligned with primary keyword intent
  • Content recommendations: gaps to fill, sections to add, topics to cover
  • Priority ranking of all detected issues by SEO impact

All fixes are stored in the Live Optimisation Table for bulk review and export.

5

Download Structured Excel Report

Click any export button to download a multi-sheet Excel workbook (.xlsx) containing:

  • Full SEO Report — every page with score, grade, all detected issues, and keyword list
  • Technical SEO Audit — per-page 0–100 scores with A–F grades and indexability status
  • Live Optimisation Table — optimised titles, descriptions, and H1s ready to paste into your CMS
  • Per-Field Issues — each issue type as a separate column for bulk CMS upload workflows
  • AI Generated Content — complete AI output for each page including all recommendations

Technical Questions

Does CrawlIQ handle JavaScript-rendered pages?

CrawlIQ uses an HTML-only crawler (aiohttp + BeautifulSoup). It reads the initial HTML response returned by the server — the same HTML that Googlebot reads on a first-pass crawl. JavaScript-rendered SPAs that return empty HTML shells will show limited content; server-side rendered or static sites get full coverage.

How long does a crawl take?

CrawlIQ runs fully asynchronous concurrent requests. A 50-page crawl typically completes in 15–45 seconds depending on your server response times. The first crawl on HuggingFace's free tier may take 30–90 seconds for the container to wake up — a cold-start notice is shown automatically.

Is my crawl data stored anywhere?

No. All crawl data is processed in an isolated in-memory session on the HuggingFace container. Data is not written to any database or disk. When a new crawl starts, the previous session is cleared. The tool is fully open-source — you can verify this in the GitHub repository.

What AI model gives the best results?

Groq (Llama 3 70B) is the fastest and works well for bulk analysis. Claude (Anthropic) gives the most nuanced content quality assessments. Gemini performs well for multilingual content. For most users, Groq's free API is the best starting point — no credit card required at console.groq.com.
